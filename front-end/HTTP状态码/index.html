http状态码

发表于 2020-04-06 | 分类于 前端 |
前后端通信中涉及到很多HTTP请求，有一些常用的状态码最好能记一下

大类

消息

1xx类型的状态码代表请求已经被接受，需要继续处理，属于临时响应。这类响应只包含状态行和某些可选的响应头，以空行结束。

  • 100 Continue 客户端应该继续发送请求。
  • 101 Switching Protocols 服务器已经理解了客户端请求，并将通过Upgrade信息头来通知客户端采用不同协议来完成该请求。
  • 102 Processing 代表处理将被继续执行。

成功

2xx类型的状态码代表请求已经成功被服务器接收、理解和接受。

  • 200 OK 请求已成功。
  • 201 Created
  • 202 Accepted 服务器已接收请求但尚未处理
  • 203 Non-Authoritative Information
  • 204 No Content
  • 205 Reset Content
  • 206 Partial Content
  • 207 Multi-Status

重定向

3xx这类状态码代表客户端需要进一步操作才能完成请求，通常用来重定向，后续的请求地址会在本次响应的location中指明。

  • 300 Multiple Choices
  • 301 Moved Permanently 被请求的资源已永久移动到新位置，将来对此资源的引用应该使用本响应返回的若干个URI之一。
  • 302 Moved Temporarily
  • 303 See Other
  • 304 Not Modified
  • ……

请求错误

4xx这类响应码代表客户端可能发生了错误妨碍了服务器的处理。除非响应的是一个HEAD请求，否则服务器应该返回一个解释当前错误状况的实体。

  • 400 Bad Request 语义有误或者参数有误，当前请求无法被服务器理解。
  • 401 Unauthorized 当前请求需要用户验证，响应中必须包含一个WWW-Authenticate信息头来询问用户信息。
  • 403 Forbidden 服务器已经请求但是拒绝执行。
  • 404 Not Found 请求的资源未在服务器上发现，最大可能是没有这个资源。
  • 405 Method Not Allowed 因为PUT,DELETE方法会对资源进行写操作，有些服务器不支持或者不允许，因此会需要这个响应码。
  • 408 Request Timeout 请求超时。客户端没有在服务器预备等待的时间内完成请求的发送。
  • ……

服务器错误

5xx类型状态码代表了服务器在处理请求过程中有错误或者异常状态发送，也可能是服务器意识到当前的软硬件资源无法完成对请求的处理。

  • 500: Internal Server Error 服务器遇到了未曾预料到的状况，通常在服务端代码bug时出现
  • 501 Not Implemented 服务器不支持当前请求所需要的某功能。
  • 502 Bad Gateway 网关或代理尝试请求时从上游接收到无效响应
  • 503: Service Unavailable 由于临时的服务器维护或过载，服务器当前无法处理。
  • 504: Gateway Timeout 网关或代理未能及时从上游接受到响应
  • ……
