  2. JS this指针

    1. 【浏览器】全局环境、对象环境、构造函数环境、DOM事件元素
      • 箭头函数的this指向父级上下文，就是要多跳一层
      • this看的是执行时的环境
    2. apply, call, bind
      • call是参数展开传进去的，apply是把所有参数合起来作为数组传的
      • bind保证obj的方法赋值给另一个变量名的时候无论在哪里都还以obj的环境运行

  10. 同步和异步

    以前浏览器里的js是单线程的，node有第三方库可以实现多线程，web worker也是另一种形式的多线程。

    原因：浏览器的多线程js读写DOM可能造成复杂的同步问题。

    问题：效率低、任务阻塞。

    同步任务：DOM渲染

    异步任务：ajax请求、定时器、静态资源加载

    1. event loop
      1. 调用栈、微任务队列、宏任务队列
      2. 先执行栈内同步任务，栈空执行微任务队列，再空执行宏任务，执行一个宏任务检查一次前面的有没有新增。
    2. promise
    3. 延时函数
      1. 属于宏任务
      2. 因为还是单线程，所以定时500ms，实际上需要等当前栈内的任务完成了，因此可能会在500ms之后才执行。
      3. 为了降低每次调用互相影响，对于连续调用，用setTimeout的递归更好
    4. 异步编程
      1. 回调函数
      2. 事件监听
      3. 观察者模式
      4. 发布订阅模式
      5. promise
      6. Generator
      7. Async/await
