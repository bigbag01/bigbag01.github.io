CDN

发表于 2020-04-04 | 分类于 前端 |
前端静态资源经常会使用CDN托管，在面试的时候也被问到一些相关的知识点。

概况

设置多个节点服务器，分布在不同区域中，便于用户进行数据传递和访问。当某一个节点出现问题时，通过其他节点仍然可以完成数据传输工作，可以提高用户访问网站的响应速度。

分布式存储

将中心平台的内容分发到各地的边缘服务器，使用户能够就近获取所需内容，降低网络用时，提高用户访问响应速度和命中率。利用了索引、缓存等技术。

负载均衡

对所有发送的请求进行访问调度，确定提供给用户的最终实际访问地址。

内容管理

负责对存储内容的监管、数据分析等

就近访问原理

这需要dns服务来进行定位，当使用dns服务的时候，可以根据他使用的LDNS服务器来进行一个定位，例如我们的调度服务器看到他是来自深圳电信的LDNS服务器，那么我们就认为该用户来自深圳电信，然后调度服务器可以让该用户去访问深圳电信的CDN服务器，这样就让用户访问到了最优的cdn节点。

用户请求文件工作过程

  1. 用户向local DNS发起请求，查找域名对应的ip
  2. localDNS迭代向rootDNS查询，逐级迭代,rootDNS=>顶级DNS=>权限DNS；
  3. 然后查询会到达CDN专用DNS服务器，它会根据用户的ip做一个全局的智能负载均衡，将最合适的cdn节点的ip返回给local DNS
  4. local DNS把这个ip返回给用户
  5. 用户向这个ip请求资源，如果有就获取，没有就会回到网站服务器源站下载发送给用户并缓存到节点。

缺陷

  1. 开发时如果断网则无法访问
  2. cdn上的库只有完整版，如果只需要使用部分，全部加载会带来冗余反而造成速度变慢
  3. 需要考虑cdn故障时的备用方案，开发工作量和复杂度高
