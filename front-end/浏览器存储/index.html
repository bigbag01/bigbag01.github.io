浏览器存储

发表于 2020-06-30 | 分类于 前端 |
HTML5出了之后浏览器多了新的存储api, 有localStorage,sessionStorage,以及web sql, indexed DB等。

浏览器存储有cookie, sessionStorage, localStorage, web SQL, indexed DB等。

共同点

保存在浏览器端，且是同源的（域名、协议、端口号相同）

特点

cookie

  • 空间：每个cookie空间不超过4K
  • 有效期默认是会话期间，可以通过设置expires来修改有效期
  • 作用域：通过domain和path确定，对于某一页面生成的cookie，同一path下的页面和子目录下的页面可以读到它。如果path设置为’/‘，作用域就是整个domain下的所有页面了。
  • 适用场景：需要请求服务器的数据
    http请求设置cookie字段可以通知服务端当前页面的域中生效的cookie
    http响应设置set-cookie字段通知客户端保存哪些cookie
  • 解析cookie

localStorage和sessionStorage

  • 空间：各有5M左右
  • 有效期 localStorage永久有效除非删除，sessionStorage的有效期是会话期间（刷新页面数据还在，关闭页面或者跳转到其他链接才会被删除）。
  • 作用域：同一浏览器不同页面相同domain可以共享localStorage，不同页面不可以共享sessionStorage
  • 存储方式：以键值对方式存储，值为字符串，可以用JSON.stringify方式将对象转为json字符串的形式，再用JSON.parse的方式获取对象。

webSQL和indexDB

前端数据库，目前只有chrome支持，用的也比较少

面试问题

localStorage如果要设置有效期要怎么设置？

  1. 封装localStorage改写set，在添加一条key-value的时候也设置expire时间，如localStorage.setItem(key,value);localStorage.setItem(key+'_expires_',expire)，在get的时候也先判断expire和当前时间，如果超时了就删除这条数据。
  2. 在constructor里先遍历所有键值，删除已过期的。

    cookie怎么设置随HTTP请求发送，怎么设置不发送？

  • 服务器的响应头加了set-cookies，客户端的http请求就会自动带上cookie
  • 服务器的响应头要设置

    如何设置cookie的有效期

    直接在cookie里添加expires条目。
