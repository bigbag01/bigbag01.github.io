前端性能优化

发表于 2020-04-04 | 分类于 前端 |
前端性能关系到用户体验，在前端性能优化方面有很多不同的角度和方法，这篇笔记做了一些整理。

优化http请求

  1. webpack打包压缩css和js文件（减少请求资源大小和次数）

  2. 用css sprites

  3. 减少本地cookie存储大小

    如果cookie中存的东西太多，当服务器来了一个带有set-cookie响应头的response，下次浏览器发送请求就需要将cookie带上。

  4. 前后端交互用json替代xml,体积小，好处理

    图像相关

  5. 使用字体或者svg替代png图标，它们是矢量图，放大不变形，且代码直接写的，渲染快

  6. 标明图像大小（这样浏览器可以直接空出它的位置，渲染其他内容，避免重排及其带来的重绘）

  7. 图像等静态资源懒加载，非首屏资源可以defer或者async方式加载

缓存

  1. 使用 CDN(content delivery network)访问静态资源
  2. 做资源本地缓存， PWA

代码优化

  1. 减少对DOM的操作，减少重绘和重排 (react和vue的vdom)
  2. <style>放在head里，<script>放到body末尾
  3. css导入减少使用@import，因为它是同步的
  4. script引入外部js时可以设置defer或者async
  5. 能用css做动画就不用js，能用原生js就不引入库

存储优化

  1. sessionStorage和localStorage做部分存储，有些数据不需要每次都向服务器请求。

其他优化

  1. web worker 用web worker线程做复杂计算避免阻塞DOM
