web worker

发表于 2020-04-04 | 分类于 前端 |
浏览器中的js是单线程的，导致在进行一些复杂运算操作的时候性能低，现在有web worker技术可以作为另外的线程帮忙做一些计算密集的任务。

作用

帮助js创造多线程环境，可以把一些计算密集任务交给web worker去做，就不会阻塞主线程，前面主线程主要负责的web ui界面就不会被卡。

注意点

文件：

  • worker通常是执行js文件,它需要与主线程的脚本同源。
  • web worker也可以嵌入html页面，主线程获取这个script转为二进制对象，再为它生成URL，主线程加载的时候就加载这个URL

环境：

worker执行时的全局对象和主线程的不同，它有自己的全局对象

  • worker不能读取主线程的DOM，不能使用document，window，parent这些对象。
  • worker能读取navigator，location对象。

通信：

  • worker和主线程直接通过消息进行通信
  • 主线程和worker之间通信的消息会被序列化再反序列化，也就是传的是值不是引用
  • 主线程和worker之间也可以传二进制数据，比如File，Blob，ArrayBuffer

使用场景

  1. 浏览器需要向服务器发起轮询，查询变化，可以把这部分工作放到worker里
  2. worker创建成功后会始终执行，不会受主线程上用户交互操作的影响被打断，但是也因此比较耗费资源。一旦任务执行完，应该立即终止worker
