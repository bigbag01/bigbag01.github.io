前端攻击及防御

发表于 2020-04-05 | 分类于 前端 |
前端面试也经常会考察到安全问题，XSS和CSRF是常见的前端安全问题。

XSS

XSS即cross-site-script跨站脚本攻击。

原理： 页面渲染的数据中包含可运行的脚本。

基本类型：反射型（url参数直接注入）和存储型（存到DB中再读出来）

XSS的可能行为

  1. 比如在url的后面加#<img src="404.html" onerror="alert('xx')">，就会让页面打开就有一个弹窗。
  2. 标签属性的值中包含引号，从而结束这个属性，插入其他内容
  3. 输入含<script>标签，在页面中插入后执行
  4. href的值中包含javascript: ,后面的内容会执行

解决方案

  1. 过滤输入内容，进行html标签转义，比如尖括号替换成&lt;&gt;,引号替换成&quot;&#39等，显示时可能会有内容混乱的问题。(一些富文本编辑插件会自带这个功能)应该尽可能在展示的时候再去做转义，因为后台的数据如果提供给移动端可能是可以直接展示的。
  2. 浏览器自带防御机制，会应对反射型攻击，http的响应头中自动会添加x-xss-protection。
  3. CSP内容安全策略指定哪些内容可执行。可以在http的meta中定义。
  4. ajax方式请求数据，js前端渲染请求内容，用innerText等api替代直接插入html
  5. 避免内联事件、拼接html

CSRF

CSRF 即 cross-site-resource-forgery，跨站请求伪造。

本质：利用用户的身份，执行非本意的操作，如在第三方网站向服务器发起请求。

例子

  1. 用户在a站前端向后端发起登录请求
  2. 后端响应确认登录成功，cookie存下用户身份认证
  3. b站前端向a站后端发起请求，带着a的cookie就可以请求成功。

漏洞在于，当前页面已经过验证，因此哪怕非本人操作也可以提交到后端。

解决方案

  1. 后端检验请求头中的referer字段，限制来自第三方的请求。
  2. 添加验证的token，django里在写html的表单的时候是有csrf_token默认值的，会和表单一起提交。这个token通常放在表单的input隐藏域和cookie中，请求提交到后端，后端可以验证请求中的token和cookie里带的是否一致。
  3. 涉及数据操作使用post而不用get，post是肯定会带referer的，get不一定带
