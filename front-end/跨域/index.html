跨域

发表于 2020-04-05 | 分类于 前端 |
跨域的产生是浏览器有同源策略，请求目标的url和当前url必须是同一domain。在本地开发浏览器访问静态页面，前后端分离独立服务器的时候经常遇到。

定义

跨域是指一个domain下的文档或者脚本去请求另一个domain下的资源，通常说的是由浏览器的同源策略限制产生的一类请求场景。

同源策略 是浏览器的一个安全功能，可以减少XSS,CSRF攻击。

同源：协议、端口、域名 都相同

解决方案

1 JSONP

原理

利用<script>这个不限制同源的标签去发请求，当请求响应到达浏览器，浏览器会分别执行对应操作。写好了callback函数之后，服务器返回的jsonp请求会在原本响应信息retval外面包上callback(retval)，当这个结果到达浏览器，就会执行具体操作。jquery ajax包装的jsonp callback执行完之后会执行success。

优缺点

  • 优点： 兼容性好，且请求结束可以通过callback传回结果。

  • 缺点：script请求实际上只能是get，不支持POST等其他http请求。只支持跨域http，不支持跨域页面的js调用。

2. CORS

由后端server配置白名单 ‘Access-Control-Allow-Origin’，如果需要带cookie的话，前后端都需要设置，前端设置writeCredentials=true。（由于同源策略限制，所读的cookie为跨域请求目标域的cookie，不是当前页面的cookie）

3. proxyTable/nginx

反向代理原理： 同源策略是浏览器的安全策略，不是HTTP协议的一部分，因此可以通过proxyTable或者nginx设置一个代理服务器，将所有请求通过代理发送。
它实际上是将请求通过一个中间层，让响应来自前端同源服务器，同时也可以修改响应的header的cookie中的domain，方便当前域的cookie写入，实现跨域登录。

4. websocket 跨域

websocket没有跨域限制，利用它来进行消息传递。

5.document.domain+iframe跨域

这种方案限制于主域相同，子域不同的情景，让两个页面都通过js强行设置document.domain为基础主域来实现同域。

6. postMessage 跨域

postMessage是HTML5的新api，它可以解决：

  1. 页面和它打开的新窗口之间的数据传递
  2. 多窗口之间的消息传递
  3. 页面和嵌套的iframe消息传递
  4. 上面三种情况的跨域数据传递。
