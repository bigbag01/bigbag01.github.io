发表于 2020-01-14 | 分类于 算法与数据结构 |
链表操作；相关题目

实现链表常用操作

记忆点

  1. 由于链表是动态数据结构，没有闲置内存，所以空间效率比数组高。但由于是动态的，无法保证内存的连续性，因此时间效率比数组低，是O(n)。

  2. 链表相关的题目经常用到的技巧

    1. 快慢指针，链表找环，或者链表找中间节点都可以用上
    2. 利用递归可以简单地翻转链表（后面有图）
    1. 递归合并两个链表

考察题目

  • leetcode - 206.reverse linked list

思路：1.用栈辅助；2.递归用一个辅助翻转函数，返回当前翻转部分头和尾，将下一个接到尾部，最后返回头。
::递归实际上可以更简单:: 不需要辅助函数来返回两个值，令head.next.next = head，就可以实现把当前节点拼到当前节点之后已翻转链表的尾部。

  • leetcode 92. reverse linked list II

思路：加个虚拟链表头，然后顺序遍历读到m和n中间的需要翻转部分，以及它之前和之后的node，参考上一题的方式翻转，再和头尾连接，返回虚拟链表头的下一个元素。

  • leetcode 138. 复制带随机指针的链表

思路：逐个复制节点，并将复制节点接到原节点之后，random也就在原节点的random之后，再提取出复制部分即可。

  • 链表中环的入口结点_牛客网

思路1：两层嵌套遍历，对每个node，遍历他们之后的list，看是否能回来。
思路2：一次遍历，对每个node，用node在内存中的地址作为hash key，node作为value,存到dict里，如果遇到已经在dict里的，那这个就是环的入口
思路3：::快慢指针::。让s和f从头出发，s每次一步，f每次两步，当他们相遇的时候让t出发，t每次一步，当s和t相遇的时候，相遇点就是环的入口。
f=2s,当fs相遇时，一定在环里，f比s多走一个周长C，因此 f-s=C => s=C. 假设入环之前的路长度为l，当s再次走到环口，路程长度应该为l+C,也就是当s和f相遇后要再走l就可以回到环口。因此这个时候让t从链表头出发，和s一起走，当t走过l到达环口的时候，就可以和s相遇，就找到了入环口。

  • 删除链表中重复的结点_牛客网

思路：设置虚拟头指针以免开头就是重复的情况，用dict记录出现过的值，如果再次出现，就删除重复节点，并用一个set对这个重复值进行记录。再遍历一次，删除值在那个set中的节点

  • leetcode 234.判断链表是否回文

    思路1：遍历列表，给每个节点加上prev指针成为双向链表，从尾到头和从头到尾两个指针一起走比较是否相等。

    思路2： 利用快慢指针找到中间节点，将后一半做翻转，对于翻转的后一半和前一半设置两个指针一起走比较是否相等。
