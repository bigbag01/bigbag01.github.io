网络通信-TCP与UDP

发表于 2020-02-24 | 分类于 网络通信
TCP和UDP的区别，三次握手，四次挥手

TCP

TCP全称是传输控制协议，它是面向连接的、可靠的、基于字节流的传输层通信协议。

  • TCP面向连接，建立连接方式是三次握手，解除连接方式是四次挥手
  • TCP只支持点对点单播传输
  • TCP不保留报文边界，以字节流传输
  • TCP是可靠的，通过段编号（seq）和确认号（ack）来判断是否丢包，如果在合理时延内未接收到确认，就重发。
  • TCP有拥塞机制
  • TCP提供全双工通信，允许双方应用程序在任何时候发送数据。

网络通信-TCP/IP模型与OSI模型

发表于 2020-02-24 | 分类于 网络通信
TCP/IP四层模型，OSI七层模型

TCP/IP模型

  • TCP/IP模型是互联网的基础，它是一系列网络协议的总称 ,比如：TCP，UDP，IP，FTP，HTTP，ICMP，SMTP 等都属于 TCP/IP 族内的协议
  • 传输控制协议/网际协议 (Transmission Control Protocol / Internet Protocol)是供已连接因特网的计算机进行通信的通信协议,它定义了电子设备（比如计算机）如何连入因特网，以及数据如何在它们之间传输的标准。
  • 通常使用的网络是在TCP/IP协议族的基础上运行的，HTTP是它一个子集.

地址公式

  • 公式一：网络地址 = IP地址 & 掩码
  • 公式二：主机地址 = IP地址 – 网络地址
  • 公式三：网络IP地址数 = 2^(32-掩码长度) （掩码长度就是掩码中连续1的个数）
  • 公式四：网络可分配地址数 = 网络IP地址数-2 （主机位全0是网络地址，全1是主机地址，去掉这两个剩下的是可分配IP地址）

URL

统一资源定位符 Uniform Resource Locator，访问web页面的地址，表示资源地点，网络上的位置。

  • URL实例符合规则scheme://host.domain:port/path/filename
    scheme通常可以是http,https,ftp,file等，http默认port为80
  • URL只能包含ASCII字符，对于非ASCII字符，使用 “%” 其后跟随两位的十六进制数来替换。
  • URL不能包含空格
