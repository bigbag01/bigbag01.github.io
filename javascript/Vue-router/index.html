Vue-router

vue-router是前端路由工具。前端路由的核心在于，改变视图的同时不会对后端发出请求。

使用router.push(‘/url’)来跳转，使用了diff算法，实现了按需加载，减少了dom的消耗

vue-router的使用实现了SPA路由的懒加载

把不同路由对应的组件分割成不同的代码块，然后当路由被访问时才加载对应的组件即为路由的懒加载，可以加快项目的加载速度，提高效率

history API

  • 传统页面跳转：使用location.href=’/url’来跳转，简单方便，但是刷新了页面；
  • HTML引入history API使用history.pushState(‘/url’)，无刷新页面，静态跳转；

hash模式和history模式

vue-router有hash模式和history模式，最直观的区别是在url有没有#

hash模式

它的特点是虽然hash出现在了URL中，但是不会被包括到HTTP请求里，对于后端没有影响，因此改变hash不会重新加载页面。

原理是window的onhashchange事件，实质上vue-router通过监听这个事件来实现路由

history模式

利用了HTML5 History Interface 中新增的pushState()和replaceState()方法，他们应用于浏览器的历史记录栈，在当前已有的back/forward/go基础上提供了对历史记录进行修改的功能。

使用场景区别

  1. hash模式下，只有hash符号之前的内容会被包含在请求里，因此对于后端，即使没有覆盖全部路由也不会报404错误
  2. history模式下，前端的URL必须和实际向后端请求的URL一致，如果后端缺少相应路由处理，会返回404错误。因此后台需要做一点配置支持，当url匹配不到任何资源的时候要返回一个这种情况的处理页面。
