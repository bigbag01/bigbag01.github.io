ES6模块化1

发表于 2019-01-23 | 分类于 javascript |
JS到ES6标准出来才有语言层面的模块管理，对于大型项目开发，模块化非常重要。

​ 模块化使得js可以将一个大程序拆分成互相依赖的小文件进行管理。在ES6之前，常用的模块加载方案有CommonJS(常用于服务器), AMD（常用于浏览器）。ES6在语言层面实现了模块功能，并且在编译时就能确定模块的依赖关系以及输入输出的变量。

关键字

​ ES6模块功能涉及的关键字主要是export和import。export用于规定模块的对外接口，import用于输入其他模块提供的功能。

特点

ES6模块设计思想是静态化，在编译时就能确定模块之间的依赖关系，而不是像CommonJS那样讲模块当做一个对象引入。

import

​ 使用import关键字可以向当前模块引入其他模块export的接口。

​ import 后在大括号中列出从外部导入的变量名，这里的变量名需要和export出来的一致。如果和当前作用域有冲突，可以用as重命名。

​ from后面是要引入的模块文件的位置，相对路径绝对路径均可。后缀如果是’.js’可省略。

​ 注意：import语句中的内容不能包含表达式和变量，因为它会在编译阶段执行，在代码运行之前。

​ 另外，执行import语句时，会去执行所加载的模块，比如import './user.js'虽然不会引入变量，但是控制台会有输出‘teacher’。

动态引入

import()

前面提到import和export都不能放在块级作用域中，也不能使用变量名或者执行语句，如果要动态加载某个模块可以用import()。import返回的是一个Promise对象。

import()接收的参数和import命令一样，它在运行时执行，import()是异步方法.
