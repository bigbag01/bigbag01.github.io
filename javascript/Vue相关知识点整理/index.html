Vue相关知识点整理

发表于 2020-04-08 | 分类于 javascript |

一、框架特性

优点

  1. 轻量级
  2. 中文文档丰富
  3. 双向数据绑定，不需要每次手动更新
  4. 组件化，页面元素可复用，视图、数据、结构分离，数据的变更不需要更改逻辑代码
  5. 虚拟DOM，不用操作原生DOM，效率高

MVVM

mvvm即Model-View-ViewModel，mvvm的设计原理是基于mvc的

  • Model代表数据模型负责业务逻辑和数据封装
  • View代表UI组件负责界面和显示
  • ViewModel监听模型数据的改变和控制视图行为，处理用户交互，简单来说就是通过双向数据绑定把View层和Model层连接起来。
  • 在MVVM架构下，View和Model没有直接联系，而是通过ViewModel进行交互，我们只关注业务逻辑，不需要手动操作DOM，不需要关注View和Model的同步工作，框架把这部分处理好了。

vue的双向绑定如何实现

vue的双向绑定是通过Object.defineProperty方法重新定义对象set和get操作来实现的。

通过使用set对data中的text这个变量做拦截，可以同时将它赋值给一个input元素的value和另一个dom元素的innerText;
使用addEventListener对input元素的keyup事件做绑定，可以在input的value发生变化的同时同步修改data的text，顺便也可以更新另一个dom元素中的内容。

二、vue生命周期

Vue实例生命周期

  1. vue实例创建

  2. beforeCreated -> 对象绑定生命周期钩子

  3. 实例初始化注入数据并校验

  4. created -> 已和data绑定，但还没有el
    判断对象是否有el选项，没有就停住直到显式调用vm.$mount(el)；
    判断是否有render函数，有就按它渲染，否则判断对象是否有template选项，没有就用el绑定的html元素中的模板，否则采用template中的；

  5. beforeMount -> 此时el也绑定了

  6. 给对象添加$el属性，替换掉挂载的DOM元素

  7. mounted -> 页面dom中的模板部分被数据替换 ==如果要操作DOM，最好在这里==

  8. 调用vm.$destroy

  9. beforeDestroy -> 此时实例还完全可用

  10. 销毁所有watcher，子组件，和事件监听器

  11. destroyed

  12. data修改 ->

  13. beforeUpdate ->

  14. 虚拟DOM重新渲染更新 ->

  15. updated -> 更新渲染完成

Vue-router 钩子函数

全局钩子

  • router.beforeEach((to, from, next) => {/}) 用来做路由控制,to,from是跳转前后的路由，next是一个必须调用的方法，可以通过next来改变跳转地址。
  • router.afterEach((to,from)=>{}),不改变导航本身

    路由配置中的钩子

    在routes中定义路由的时候可以给每个加上beforeEnter的路由钩子，和全局的beforeEach用法差不多，只不过是针对指定路由的

    路由实例中的钩子

  • beforeRouteEnter(to,from,next)，还不能通过this访问到当前组件，可以通过next(vm=>{})来做些操作。
  • beforeRouteUpdate(to,from,next)，可以访问当前组件了，next不支持回调
  • beforeRouteLeave(to,from,next) ，可以通过next(false)来阻止离开

三、Vue 工具和插件

Mixin

引入组件之后，和组件进行合并，向组件添加方法和对象

jquery ajax, axios, fetch

传统ajax是对XHR的封装，请求之间有依赖关系容易出现回调地狱。

  • jquery是之前MVC模式开发时流行使用的库，如果只是需要ajax功能引入整个jquery比较没必要，如果层层依赖容易写出回调地狱
  • axios是基于promise实现的http库,vue推荐和它结合使用。支持promise API(then,catch,finally)，客户端支持防止CSRF(cross site request forgery)
  • fetch不是对XHRHttpRequest对象的封装，使用了Promise，它更底层，API丰富，但是默认不带cookie，不能监测请求进度，不支持超时控制，对于响应码中的400、500都会当成成功
  • 总的来说*， 现在axios是前端比较合适的请求工具，和vue配合用的比较多。

vue-loader

  • vue-loader负责解析.vue文件，提取其中的script,style及html template然后分别转给各个loader去处理，最后组装成一个commonjs模块。
  • 使用vue-loader，可以在vue文件中使用ES6语法，style可以用less，sass等，html模板可以用jade。

babel

  • babel-preset-env做的是js的编译，把新引入的ES语法转为ES5的语法，可以在.babelrc文件里配置目标适配情况，从而能适配各浏览器。
  • ES6新引入的对象，比如Set，Promise，要通过polyfill来解决
  • Polyfill 是一块代码（通常是 Web 上的 JavaScript），用来为旧浏览器提供它没有原生支持的较新的功能

四、开发细节

封装vue组件的过程

  1. 确认这个组件的作用，是否需要封装复用
  2. 确认这个组件的交互逻辑（是否需要和外部组件交互，数据接口）
  3. 建立组件的template，写基本样式
  4. 设计组件的输入，props里的数据及类型
  5. 设计组件需要暴露的方法

params和query

this.$route.query，类似get的参数，存在于url中，刷新不丢失；
this.$route.params，类似post的参数，不显示出现在url中，刷新丢失。

父子组件通信

父传子

  • 静态内容，父组件在子元素直接写属性名和值
  • 动态绑定，父组件在子元素中写v-bind:msg=”message”，message可以通过v-model和表单元素绑定，也可以绑定父组件data中定义的变量
  • 子组件通过props来获取

    子传父

  • 父组件在子元素中定义监听事件<son-component @update-data="update" > </son-component>，在methods中定义update方法来处理监听到的事件。
  • 子组件通过this.$emit('update-data',msg)方法向父组件传递信息。

v-show和v-if

二者都可以控制组件是否显示，但v-show=false实际做的是设置元素的display:none,还可以切回显示状态，切换状态的开销比较小。而 v-if=false则是控制这个元素是否要添加到dom树中，如果设为false后面都不能再显示，如果要切换，开销比较大。

css scope

在当前页面的<style>标签中加scoped就可以保证这部分css只会影响当前的组件

<keep-alive>作用

使被包含的组件保持状态，避免重新渲染

this.$refs.domName

vue常用指令

  1. v-model 创建双向数据绑定，通常用在表单元素上（实际上做的事：1. v-­bind 绑定一个 value 属性 2.v-­on 指令给当前元素绑定 input 事件）
  2. v-bind 使后面的值绑定一个动态变量，不绑定则是常量
  3. v-for循环
  4. v-if条件控制是否显示
  5. v-on事件绑定
