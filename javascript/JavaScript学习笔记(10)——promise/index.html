Javascript学习笔记(10)——promise

发表于 2020-02-14 | 分类于 javascript |
ES6新引入的promise,主要用于异步编程，有点复杂，但是好用

概念

Promise对象可以看成一个容器，保存一个未来执行的事件的结果。从Promise对象中可以获取异步操作的消息，方便异步任务的处理。

特点：

  1. Promise代表一个异步操作有三种状态：pending(进行中),fulfilled(成功了),rejected(失败了)
  2. 一个异步操作的结果决定了状态之后，状态不会改变。状态只会从 pending -> fulfilled或者pending -> rejected，当这个状态确定之后就变成了resolved
  3. promise一旦建立就无法取消
  4. promise内部错误如果不做处理不会向外传
  5. pending状态无法得知进展如何

用法

基本写法

  • resolve和reject是两个js引擎给定的函数
    • resolve用来将promise对象的状态从pending转为fulfilled，把得到的结果value传出去，之后then接收
    • reject则是把状态转成rejected，把错误信息err传出去，之后catche接收
    • finally是在then或者catch之后执行，不接受参数
  • promise构造时内部的代码是立刻执行的
  • resolve和reject传出的参数就是then执行时接收的参数，这个参数也可以是一个promise,如果这样的话，实际上当前promise的then等待的状态是依赖于被传回的promise的。

promise组合

  • 由多个promise all组合得到一个promise，这个的状态依赖于所有参数，只有当大家都fulfilled，这个才会resolve，否则就reject，其中一个reject了就直接reject
  • allSettled是等所有promise都返回结果，无论是fulfilled还是rejected，才结束。
  • race则是返回第一个结束的promise的状态作为结果
  • any就是只要有一个的结果是fulfilled就会resolve

链式调用

then中返回的值val会被包装成Promise.resolve(val)，从而被下一个then调用

resolve方法

resolve会把参数包装成一个promise对象

reject方法

reject方法的参数要求是一个thenable对象，catch中获得的参数不会被包装，直接会是这个thenable对象。
reject方法的返回值是一个promise对象，状态为rejected
