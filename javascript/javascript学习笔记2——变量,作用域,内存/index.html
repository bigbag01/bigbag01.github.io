Javascript学习笔记(2)——变量，作用域，内存

发表于 2017-11-08 | 分类于 javascript |

变量

​ ECMAScript 变量有两种不同数据类型的值：基本类型值（简单的数据段）和引用类型值 （可能由多个值构成的对象）。

​ 基本数据类型是按值访问的，可以操作保存在变量里的值。

​ 引用类型的值是保存在内存里的对象，是按引用访问的。操作对象时，是操作对象的引用。

​ 注意： ECMAScript和其他语言不同，这里的字符串是基本类型而非引用类型

属性创建

  • 引用类型可以动态添加或删除属性
  • 基本类型的值不能添加属性

变量复制

  • 基本类型值的复制就是创建一个副本，创建之后二者互不干扰，占据各自内存空间
  • 引用类型变量的复制是创建一个指针的副本，二个变量指向内存中同一个位置，改变其中一个变量，另一个也被改变。

参数传递

​ ECMAScript中所有参数都是按值传递的。

  • 在向函数传递引用类型的值的时候，会把这个值在内存中的地址复制给一个局部变量，因此它的变化会反映到函数外部。

  • 函数参数为引用类型的时候，也是按值传递的。

    B是对A进行复制得到的，它得到了和A指向的同一个对象的地址。

    第3行，在foo函数内，B和A指向了内存中的同一个对象，对B的属性进行操作，也就是对A的属性进行操作，因此最后输出A.name是b

    第4行，对B进行了重新赋值，在内存中开辟一个新空间给一个新的变量，而它是一个局部变量，在函数执行完毕后会被销毁。这个时候B和A已经指向两个地址，因此对B的修改不会影响到A

类型检测

  • 用typeof操作符可检验变量的数据类型，具体可见前一篇博客。

  • 检验变量是否是给定引用类型的实例可以用instanceof

执行环境和作用域

​ 函数的每次调用都有与之紧密相关的作用域和执行环境。从根本上来说，作用域是基于函数的，而执行环境是基于对象的。
​ 换句话说，作用域涉及到所被调用函数中的变量访问，并且不同的调用场景是不一样的。执行环境始终是this关键字的值，它是拥有当前所执行代码的对象的引用。每个执行环境都有一个与之关联的变量对象，环境中定义的所有变量和函数都保存在这个对象中。虽然我们编写的代码无法访问这个对象，但解析器在处理数据时会在后台使用它。

全局执行环境

全局执行环境是最外围的一个执行环境，根据宿主环境不同，表示执行环境的对象也不一样。

  • web浏览器中全局执行环境是window对象
  • nodejs中全局对象是global，global的属性可以直接访问到
  • 全局执行环境直到应用程序退出（浏览器页面关闭）才被销毁

局部执行环境

每个函数都有自己的执行环境，当其中代码执行完毕后，该环境被销毁。

作用域链

  • 当代码在一个环境中执行，会创建变量对象的一个作用域链。

  • 作用：保证对执行环境有权访问的所有变量和函数的有序访问。

  • 作用域链顺序：作用域链最前是当前执行的代码所在环境的变量对象，向后依次为外一层包含环境，一直延续到全局执行环境。

    标识符解析沿作用域链一级一级向后搜索。

    内部环境可以通过作用域链访问外部环境，但反之则不可。

延长作用域链

  • try-catch语句的catch块

    创建一个新的变量对象，其中包含被抛出的错误对象的声明

  • with语句

    将指定的对象添加到作用域链中

块级作用域
  • 在ES6之前，js中没有块级作用域，使用var声明的变量会被添加到最接近的环境中。
  • ES6提供了块级作用域，在strict模式下，通过let或const来体现。
变量提升

​ js中会把定义在后面的东西（函数或者变量）提升到前面定义。

​ 虽然在函数中 alert语句之前a还未被声明，但由于js中的变量提升，实际变量a在函数一开始就被声明，但未被赋值。对于变量h，局部作用域中的h被提升，和全局变量h比，它在作用域链中更靠前，因此输出的是局部作用域中的h的值，由于赋值语句在之后，此时的输出也是undefined。

​ 对于函数提示，函数声明方式可以提升，而函数表达式则不能

内存

js中有**自动垃圾收集机制**，执行环境会负责管理代码执行过程中使用的内存。

​ 这种垃圾收集机制的原理是：找出那些不再被使用的变量，释放其内存。垃圾收集器会按照固定的时间间隔周期性地执行这个操作。

垃圾收集策略

​ 离开作用域的值会自动被标记为可回收，在垃圾收集期间会被删除。标记清除为目前主流策略，另一种是引用计数。当代码中存在循环引用现象，“引用计数”算法会出问题。

内存管理与性能优化

​ 确保占用最少的内存可以让页面获得更好的性能。而优化内存占用的最佳方式是i未执行中的代码只保存必要数据。一旦数据不再有用，就将其设为null来解除引用。主要适用于全局变量及其属性。 解除引用不等于自动回收该值所占内存，实际是让这个值脱离执行环境，以便垃圾收集器下次运行时将它回收。
