Javascript学习笔记(2)——变量，作用域

发表于 2017-11-08 | 分类于 javascript |

变量

​ ECMAScript 变量有两种不同数据类型的值：基本类型值（简单的数据段）和引用类型值 （可能由多个值构成的对象）。

​ 基本数据类型是按值访问的，可以操作保存在变量里的值。

​ 引用类型的值是保存在内存里的对象，是按引用访问的。操作对象时，是操作对象的引用。

​ 注意： ECMAScript和其他语言不同，这里的字符串是基本类型而非引用类型

属性创建

  • 引用类型可以动态添加或删除属性
  • 基本类型的值不能添加属性

变量复制

  • 基本类型值的复制就是创建一个副本，创建之后二者互不干扰，占据各自内存空间
  • 引用类型变量的复制是创建一个指针的副本，二个变量指向内存中同一个位置，改变其中一个变量，另一个也被改变。

参数传递

​ ECMAScript中所有参数都是按值传递的。

  • 在向函数传递引用类型的值的时候，会把这个值在内存中的地址复制给一个局部变量，因此它的变化会反映到函数外部。

  • 函数参数为引用类型的时候，也是按值传递的。

    B是对A进行复制得到的，它得到了和A指向的同一个对象的地址。

    第3行，在foo函数内，B和A指向了内存中的同一个对象，对B的属性进行操作，也就是对A的属性进行操作，因此最后输出A.name是b

    第4行，对B进行了重新赋值，在内存中开辟一个新空间给一个新的变量，而它是一个局部变量，在函数执行完毕后会被销毁。这个时候B和A已经指向两个地址，因此对B的修改不会影响到A

类型检测

  • 用typeof操作符可检验变量的数据类型，具体可见前一篇博客。

  • 检验变量是否是给定引用类型的实例可以用instanceof

执行环境和作用域

作用域概念

  • JS中有三种作用域：全局作用域，函数作用域，块级作用域
  • 块级作用域由{}包括

全局执行环境

全局执行环境是最外围的一个执行环境，根据宿主环境不同，表示执行环境的对象也不一样。

  • web浏览器中全局执行环境是window对象
  • nodejs中全局对象是global，global的属性可以直接访问到
  • 全局执行环境直到应用程序退出（浏览器页面关闭）才被销毁

变量声明关键字 var, let, const

var

  • var定义的变量没有块的概念，如果在函数中定义，它的作用域就是函数作用域，不能跨函数访问，如果在全局定义，就是全局变量。
  • 浏览器环境中，var声明的全局变量是window对象的属性

    let

  • let定义的变量只能在它所在的块作用域中访问，不能跨函数访问
  • 浏览器环境中，let声明的全局变量不是window对象的属性
  • 在相同作用域或者块级作用域中，不能用let重新声明var或let声明过的变量，不能用var重新声明let声明过的变量，但是可以用var重置var声明过的变量

    const

  • const定义常量只能在它所在块作用域中起作用，不能跨函数访问，不能修改，必须有初始化赋值。
  • const 定义的常量是一个引用，如果定义的是一个对象，实际可变，但不可以再赋值

注意：const和let声明的全局变量不属于顶层对象的属性

作用域链

  • 当代码在一个环境中执行，会创建变量对象的一个作用域链。

  • 作用：保证对执行环境有权访问的所有变量和函数的有序访问。

  • 作用域链顺序：作用域链最前是当前执行的代码所在环境的变量对象，向后依次为外一层包含环境，一直延续到全局执行环境。

    标识符解析沿作用域链一级一级向后搜索。

    内部环境可以通过作用域链访问外部环境，但反之则不可。

延长作用域链

  • try-catch语句的catch块

    创建一个新的变量对象，其中包含被抛出的错误对象的声明

  • with语句

    将指定的对象添加到作用域链中

变量提升

JavaScript 中，var 关键字定义的变量可以在使用后声明，也就是变量可以先使用再声明

变量声明会提升，变量初始化不会提升

函数提升

  • 函数声明可以被提升，但是函数表达式相当于变量初始化，不会被提升。

  • js在预编译的时候，总是优先处理函数声明，然后才是函数表达式
