js的垃圾回收机制

内存

js中有自动垃圾收集机制，执行环境会负责管理代码执行过程中使用的内存。
这种垃圾收集机制的原理是：找出那些不再被使用的变量，释放其内存。垃圾收集器会按照固定的时间间隔周期性地执行这个操作。

垃圾收集策略

标记清除为目前主流策略，另一种是引用计数（现代浏览器不太用了）。

标记清除

  1. 当一个变量进入内存环境，就带上标记
  2. 垃圾回收器定时检查，给所有当前环境中的变量及他们引用的变量去掉标记
  3. 删除那些还有标记的变量，释放内存

内存管理与性能优化

确保占用最少的内存可以让页面获得更好的性能。而优化内存占用的最佳方式是i未执行中的代码只保存必要数据。一旦数据不再有用，就将其设为null来解除引用。主要适用于全局变量及其属性。 解除引用不等于自动回收该值所占内存，实际是让这个值脱离执行环境，以便垃圾收集器下次运行时将它回收。

JS程序堆栈

堆

堆用于复杂数据类型分配空间，比如数组、object，运行时动态分配内存，存取速度慢
闭包中的变量保存在堆中

栈

存放基本类型变量和对象的引用，存取速度快。
栈是后进先出
