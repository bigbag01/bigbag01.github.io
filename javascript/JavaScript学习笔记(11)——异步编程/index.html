Javascript学习笔记(11)——异步编程

发表于 2020-02-15 | 分类于 javascript |
ES6新引入了一些特性之后，异步编程有很多种不同写法和实现，这里主要也是参考别人的博客还有文章整理的一些笔记。

单线程的JS

js是单线程的,(nodejs可以通过第三方库来实现多线程)
原因：多线程同时读写DOM会造成复杂的同步问题
结果：如果任务全都同步进行，则同一时间只能处理一件任务，如果前一项时间过长（ajax，IO…）会阻塞后面任务的进行，因此还需要异步任务。

  • 典型的同步任务：DOM元素的渲染
  • 典型的异步任务： ajax请求，setTimeout定时器, 图片、音乐的加载

异步机制

JS的异步是通过任务队列来实现。异步任务先做自己的事，比如向服务端发请求等待响应，等它完成会向任务队列添加一个事件，表示它完成了。当异步任务开始的时候，主线程可以做其他工作，当它空下来就会去读任务队列里的完成事件，如果某个事件指定了回调函数，那主线程就会执行这个回调，也就完成了异步任务。

异步实现

1. 回调函数

最经典的jquery包装的ajax写法

这种写法如果遇到多个回调函数需要嵌套依赖就比较糟糕，并且不能直接返回值。

2. 事件监听

在全局注册事件‘done’，当f1执行完后触发该事件，监听者就会做出对应操作。
优点：对于一个事件可以添加多个事件监听，也就是可以有多个回调函数，相比于嵌套的回调来说，可以去耦合，有利于实现模块化。
缺点： 程序变成事件驱动型，流程不清晰。

3. 观察者模式

观察者模式是一个对象Subject维护一系列依赖于它的对象Observer，当自己发生状态变化的时候，就通知他们也更新。

注意： Subject是可以添加、删除、通知Observer更新的，二者之间的联系是直接的。
Vue的双向绑定用的是观察者模式

4. 发布订阅

publisher通过发布主题事件的方式通知订阅这个主题的subscriber，subscriber对一个主题通过自定义事件来订阅。

注意： 发布订阅和观察者不一样的，观察者是被观测者直接维护观测对象，自己变了通知他们，发布订阅是订阅者订阅主题，发布者把消息按主题发给中间件，有消息的时候订阅者可以随之做出操作。

有await，且返回值是一个promise就会等它返回再往下走，如果返回值不是promise会直接往下走，无法保证结果的顺序。
