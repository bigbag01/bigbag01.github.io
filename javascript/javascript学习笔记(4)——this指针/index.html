Javascript学习笔记(4)——this指针

发表于 2020-02-05 | 分类于 javascript |

一、不同环境下的this指向【浏览器】

全局环境

浏览器的全局环境，this指window对象

对象环境

对象环境，this指向该对象

实际this指向和函数运行环境有关。

  • say3虽然是obj.say2的复制，但是运行的环境是window，实际执行的是window.say3()，因此输出的env是out.
  • obj.say4()会返回一个函数，因此需要再调用，因为返回的函数在全局环境了，因此输出的是out。
  • obj.say5是一个箭头函数，箭头函数中的this指向父级的上下文，也就是obj的上下文，也就是window，因此输出inside
  • obj.say6是匿名执行的箭头函数，箭头函数的上下文相当于这个匿名函数的上下文，执行它的是obj，因此输出的是inside

构造函数环境

构造函数环境，this指向构造出的对象实例。用new调用构造函数的时候，会产生一个空对象，然后用call将构造函数中的this指向这个对象。执行完之后，这个空对象就有了相关的属性。构造函数不需要返回值。如果有返回值且是一个对象类型，那构造出的实例的this指向构造函数返回的这个对象。

DOM事件对象环境

DOM事件环境，哪个元素触发的，this就指向谁

二、用apply，call，bind修改this

apply 和 call，修改函数运行时的上下文

js中的call和apply都是用于修改函数运行context的。

  • call 和apply作用相同。传进第一个参数obj,使它成文执行say的上下文, 相当于 obj.say=say1;obj.say()
  • call和apply的区别在于函数参数的传递，call是将函数调用参数依次写在第一个参数之后，apply则是将它们放进数组，放到第一个函数后。
  • 当传入的第一个参数为null时，函数内的this指向全局环境，否则指向传入的对象。
  • 在构造函数执行时，this指向当前构造函数构造的对象实例，因此，在执行new Patient()的时候，传递给Person.call的this是p

bind将作用域与函数绑定

  • bind保证了无论函数在什么上下文执行，this都指向所需指向的对象

apply,call与bind的区别

  1. apply和call作用相同，只是参数传递方式不同，call是一个一个传，apply是传数组
  2. apply和call是对于给定的context，立即调用函数并返回结果。
  3. bind是对于给定的context，绑定函数和这个context，并返回绑定后的函数，不执行该函数。
