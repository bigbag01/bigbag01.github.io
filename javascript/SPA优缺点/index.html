SPA优缺点

发表于 2020-04-09 | 分类于 javascript |

SPA

Spa 即single page application.一个spa就是一个web应用，所有的静态资源一次加载完成。在交互的时候由前端路由动态载入，单页面跳转的时候相当于是组件的刷新，所以实际上只刷新局部，PC端用的多。多页应用的页面跳转是整页刷新。

优缺点

SPA优点

  1. 减轻服务器端压力，服务器先将一份包含了静态资源、js及模板的数据发送到了客户端，之后客户端只要请求少量的渲染页面或视图所需的数据就可以了。

  2. 因为每次页面切换一部分组件不变，因此只需要渲染局部，页面切换快，用户体验好。

  3. 前后端分离，开发快速。后端可以只开发统一的http api供pc、app等不同客户端使用，将渲染工作交给前端。前端不需要带着后端工程一起调试，可以用mock api的方式直接进行开发和调试。最后再联调。

  4. 前后端分离开发可以独立部署，分别做水平扩展和负载均衡。后端挂了前端也还是可以展示部分内容。

    P.S.我们前后分离开发的时候用faker.js配合json-server作为mock工具。

SPA缺点

  1. 初次加载耗时多；
    1. 将一部分静态资源设置为懒加载，如非视区内的图片等页面拉到该区域再加载。
    2. 将一部分静态资源部署到cdn
    3. vue3.x的懒观察可以减少渲染前初始化观察者的时间
  2. 不利于SEO（search engine optimization），难以被搜索引擎发现。
    1. 由于页面数据都是后续ajax加载，大多数搜索引擎的爬虫不会等异步执行结束再抓数据，因此SPA模式下SEO的效果会很差。
    2. 可以利用Server-Side Rendering 来解决。将SPA应用打包到服务器上，在服务器上渲染HTML并插入部分meta信息，发送到浏览器，再和SPA框架配合再浏览器上混合成为可交互的应用。（框架如vue的Nuxt和React的Next）
  3. 导航需要单独维护
    1. vue可以用vue-router
