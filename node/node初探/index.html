node简介

发表于 2017-12-21 | 分类于 javascript ， node |

node是什么？

​ 类似于chrome，Node 是基于事件驱动的异步架构。浏览器通过事件驱动来服务界面上的交互，node通过事件驱动来服务I/O。

node的特点？

  • 异步I/O

  • 事件与回调函数

    轻量级、松耦合、只关注事物点

  • 单线程

    浏览器中的js也是单线程。

    • 优势 不用考虑多线程状态同步问题，没有线程上下文交换的开销。
    • 弱点 1.无法利用多核CPU; 2.错误会引起整个应用退出; 3. 大量计算占用CPU会导致无法继续调用异步I/O
    • 解决大计算量方案 child_process

  • 跨平台

    基于libuv实现

node面对不同场景

I/O密集型（极擅长）

​ node面向网络并且擅长并行I/O，Node利用事件循环的处理能力，而不是启动每一个线程为每一个请求服务。

CPU密集型（不擅长，可解决）

​ 由于js单线程的原因，如果有长时间运行的计算，会导致CPU时间片无法释放，使得后续I/O无法发起。

解决方案
  • 编写C/C++扩展来高效利用CPU
  • 通过子进程方式，将一部分计算任务分配给子进程，让子进程负责计算，通过进程间消息传递把结果返回给父进程。（听着比较复杂，但是对于CPU密集型应该是能够比较好的解决的）

node应用

  • 开发跨平台桌面应用 。nodejs调用本地API，html+css+javascript编写页面，可以直接构建桌面应用。成熟的解决方案有node-webkit (NW.js)和electron
    • 二者比较——来自知乎
    • electron官网
  • 高性能I/O用于实时应用。 提供实时功能，如实时语音、实时通知。常用的工具：socket.io
  • 并行I/O用于分布式环境
